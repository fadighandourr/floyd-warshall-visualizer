Floyd-Warshall Algorithm

The Floyd-Warshall algorithm is a classic all-pairs shortest path algorithm. Unlike Dijkstra's algorithm — which finds the shortest path from a single source to all other nodes — Floyd-Warshall computes the shortest distances between every pair of vertices in a weighted graph, all at once.

It was published by Robert Floyd in 1962 and is closely related to work by Stephen Warshall on transitive closure of graphs. The algorithm works on both directed and undirected weighted graphs, and it handles negative edge weights — as long as there are no negative-weight cycles.

Dynamic Programming All-Pairs Shortest Path Weighted Graphs Supports Negative Edges O(V³) Time

The core idea is beautifully simple: for every pair of nodes (i, j), check whether routing through an intermediate node k gives a shorter path than the current known one. This check is repeated for every possible intermediate node, and after all iterations the matrix holds the globally optimal shortest distances.

Key recurrence:
dist[i][j] = min( dist[i][j], dist[i][k] + dist[k][j] )

The matrix is initialized with direct edge weights (or ∞ if no edge exists, and 0 on the diagonal). After running all three loops, dist[i][j] holds the length of the shortest path between every pair of vertices i and j.

To detect negative cycles, check the diagonal after the algorithm finishes — if any dist[i][i] < 0, a negative cycle exists in the graph.

Real-World Uses of Floyd-Warshall

Any problem that needs shortest distances between all pairs of nodes is a natural fit.

🗺️ GPS & Routing Systems

Road networks use all-pairs shortest paths to pre-compute travel times between major hubs, enabling instant route suggestions for any origin–destination pair.

🌐 Network Routing Protocols

In computer networks, Floyd-Warshall (or its variants) helps routers build routing tables that know the optimal path between any two nodes in the network topology.

✈️ Airline & Transit Planning

Airlines and transit systems use all-pairs shortest paths to find the cheapest or fastest connections between any two cities, including multi-stop itineraries.

🤝 Social Network Analysis

Measures like betweenness centrality and graph diameter require shortest paths between all node pairs — Floyd-Warshall makes this computation straightforward.

🎮 Game & AI Pathfinding

In game maps or robot navigation grids, pre-computing all-pairs distances allows instant lookup for any agent movement decision without re-running search each time.

📦 Supply Chain & Logistics

Optimizing warehouse-to-warehouse or supplier-to-factory routes benefits from knowing the minimum cost path between every pair of locations in the distribution network.

🔗 Transitive Closure

A Boolean variant of Floyd-Warshall determines reachability: can node i reach node j? This is used in compilers, database query optimization, and dependency analysis.

💹 Arbitrage Detection (Finance)

By applying Floyd-Warshall to currency exchange rate graphs (using log weights), traders can detect arbitrage opportunities — cycles where exchanging currencies yields profit.

When NOT to use Floyd-Warshall:
  • When you only need a single-source shortest path → use Dijkstra (faster for sparse graphs).
  • When the graph is very large and sparse → O(V³) becomes too slow; prefer repeated Dijkstra or Johnson's algorithm.
  • When the graph contains a negative-weight cycle → the algorithm cannot produce correct results.

How It Works

We check if going through an intermediate node k improves the shortest path between nodes i and j. 

d[i][j] = min(d[i][j], d[i][k] + d[k][j])

Complexity (CLO-1)

Time: O(V³)

Space: O(V²)

Three nested loops (k, i, j) → cubic complexity.

Dynamic Programming (CLO-3)

The algorithm reuses previously computed shortest paths, making it a Dynamic Programming approach.
